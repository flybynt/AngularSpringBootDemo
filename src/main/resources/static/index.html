API / MVC Demo

This web full stack demo uses a REST API generated with Springboot using an H2 embedded database, which exposes the API using two popular Springboot approaches. The presentation portion of the demo is composed of an MVC Angular application that is intended to demonstrate the principals of client based MVC, responsive design, and accessibility semantics. The demo UI was tested with an LG3 smartphone, Kindle Fire tablet, and HP laptop using Chrome browser using both responsive and device simulation modes.

The demo also demonstrates various methods to register event notifications that the current page has loaded which can be used to bootstrap UI and app functionality. These techniques and or careful placement of scripts tags are used to ensure that the user experience is not greatly impacted during app loading and rendering. Processing of these event registration and notification techniques is demonstrated on this page.

The demo app can be accessed as individual page based applications or as a SPA (Single Page Application) utilizing angularJS routing. The same angularJS controllers are shared for both approaches.

Development Related Requirements

The primary requirements for the demo MVC app and REST API were set forth as follows

User Interface

  • Build a form that takes the customer name, email address and some free form comments
  • Must be a client side MVC based app (AngularJS or React)
  • Think about page HTML quality semantics (simplicity, page size, field names etc)
  • Apply some CSS to the HTML to allow for responsive design and common rendering across mobile devices and screen sizes
  • Show modularization of JS and framework code
  • When implementing form(s), ensure add some validation and field status
  • Ensure the page is developed with attention for Accessibility in mind for the disabled

REST API

  • Build a back-end REST API on the that receives the comments and info from the MVC app and writes them to sys out (obviously not production ready but for demo purpose!)
  • The API should validate comments, remove malicious characters as necessary before storing them
  • The API should implement some pre-defined max lengths (200 chars for the comments and 100 for other fields)
  • Email entry format semantics should be enforced for email entries so that they are validated before being stored
  • The API should respond with appropriate http SUCCESS and ERROR codes and messages which should be able to be written to the browser console by the MVC app
  • The end-points should be able to be accessed via clients other than your MVC app such as curl or any web browser
